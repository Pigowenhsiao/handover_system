## ADDED Requirements
### Requirement: レイアウトの一貫性
システム SHALL 各主要ページで同一の構造と余白（タイトル/内容/操作/状態）を採用し、ユーザーが迅速に識別/操作できるようにする。
モダン UI のスタイルとレイアウトに合わせて全ページを再配置する。

#### Scenario: 主ページと管理ページの統一
- **WHEN** ユーザーが異なるページへ切替
- **THEN** タイトル表示、内容領域、操作ボタンの並びが同一ルールで維持される

### Requirement: 入力欄の視認性
システム SHALL Entry/Combobox/Text の入力欄に明確な枠線を表示し、入力領域の境界を判別できるようにする。

#### Scenario: テキスト入力欄の枠線
- **WHEN** ユーザーが日報/設備異常/異常ロットの入力欄を見る
- **THEN** テキスト入力欄は枠線が表示され、境界が視認できる

### Requirement: 多言語文言の一貫性
システム SHALL すべての可視文言を言語リソースキーで提供し、日/中/英の翻訳が揃っていることを保証する。

#### Scenario: 文言ソースの統一
- **WHEN** UI 上にボタンやラベルが表示される
- **THEN** 文言は言語リソースから取得され、ハードコードされない

### Requirement: 言語切替の即時更新
システム SHALL 言語切替時に全画面の文言（サイドバー、タイトル、ステータス、タブ）を即時更新する。

#### Scenario: 言語切替後の更新
- **WHEN** ユーザーが言語を切替
- **THEN** すべての可視文言が選択言語に即時更新される
- **WHAT** 言語切替後も現在ページに留まり、主画面に戻らず文言のみ更新される

### Requirement: 操作状態の明確化
システム SHALL ログイン状態に応じて機能の可用性と状態表示を調整し、誤操作を防止する。

#### Scenario: 未ログイン時の管理制限
- **WHEN** ユーザーが未ログイン
- **THEN** 管理機能は無効で、ステータスに未ログインを表示する
