# 利用者マニュアル

## 1. ログインと言語/テーマ
- 起動するとログイン画面が表示され、ログイン前に言語（日/中/英）を切替できます。
- ログイン後、右上の言語セレクターで言語を切替できます。
- 主画面にはライト/ダークのテーマ切替があり、即時反映されます。

## 2. 日報と基本情報
- 日付・シフト・エリアを選択（日付はカレンダー選択）。
- 基本情報を保存すると、他の機能ページが解放されます。
- 同一 日付 + シフト + エリア は同一日報として扱い、再保存で日報主ファイルと出勤データが上書きされます。

## 3. 出勤記録
- 正社員/契約社員の定員・出勤・欠勤・理由を同一ページで入力。
- 保存後は当該日報に関連付けられます。

## 4. 設備異常 / 異常ロット
- 設備異常：複数記録を追加（設備番号、異常内容、発生時刻、影響数、対応内容）。
- 異常ロット：複数記録を追加（ロット番号、異常内容、処置状況、特記事項）。
- 保存後は当該日報に紐づきます。同日報への再保存では設備/ロットは累積で保持されます。

## 5. 出勤統計（総結）
- 統計開始/終了日を選択（既定は当月1日〜今日）し、「確定」で集計。
- 出勤統計表（日時/エリア/記入者/正社員/契約社員/備考）を表示。
- 出勤率折線図と出勤人数の積み上げ棒グラフを表示。

## 6. 異常履歴照会
- 日付範囲とシフト/エリアを選択して検索。
- 設備異常と異常ロットの二表を表示（入力詳細を含む）。
- 画面を超える場合はスクロールして全体を確認できます。

## 7. Delay List / Summary Actual
- 日付範囲で検索。
- Excel インポート → 仮保存 → アップロードの手順。
- テーブル上で編集・削除が可能。アップロード前に内容確認を推奨。

## 8. 注意事項
- すべての日付はカレンダー選択を推奨し、手入力ミスを避けます。
- アップロード前のデータは DB に書き込まれません。
- `data/handover_system.db` と `handover_settings.json` の定期バックアップを推奨します。
