# 調査レポート (Main Spec Research)

## 1. 技術候補の比較

### データベース
- SQLite：単機アプリに最適、運用コストが低い
- PostgreSQL/MySQL：高機能だが運用コストが高い

### GUI
- Tkinter：標準搭載、軽量
- PyQt：高機能だが配布コスト増

### 多言語
- JSON リソース：管理が容易、即時切替に適合
- gettext：標準だが構成が複雑

## 2. 採用理由
- 単機運用を前提とし、簡潔で保守しやすい構成を優先
- パッケージ配布しやすい技術スタックを選択

## 3. 重要ポイント
- 日報は「日付+シフト+エリア」で一意
- 基本情報未保存時は他機能をロック
- 出勤統計と異常履歴は期間指定が必須
- Delay List / Summary Actual は仮保存後にアップロード

## 4. PyInstaller
- ローカル環境での実行ファイル化に有効
- 設定は `handover_modern.spec` に固定
