# 多言語機能 エンドツーエンドテスト仕様

本システムは単機デスクトップアプリのため、テストは手動フローを中心とする。

## テストシナリオ 1: 言語選択と切替
**目的**: 言語切替と即時更新を確認

**手順**:
1. アプリを起動し、既定言語が日本語であることを確認。
2. ログイン画面で中文に切替。
3. ログイン画面の文言が即時更新されることを確認。
4. ログイン後に英語へ切替。
5. 主画面の各領域の文言が即時更新されることを確認。

**期待結果**:
- 言語切替が成功し、クラッシュしない
- タイトル/ボタン/ヘッダが正しい言語で表示される

## テストシナリオ 2: 切替時の状態保持
**目的**: 言語切替が入力内容に影響しないことを確認

**手順**:
1. 出勤ページで一部数値を入力。
2. 言語を切替。
3. 入力値が維持されていることを確認。

**期待結果**:
- 入力状態が保持される
- 文言のみ更新される

## テストシナリオ 3: 翻訳リソースの入出力
**目的**: 管理者が翻訳をインポート/エクスポートできることを確認

**手順**:
1. 管理者として「翻訳リソース管理」を開く。
2. JSON をインポート。
3. エクスポートし内容を確認。

**期待結果**:
- インポート/エクスポートが成功
- JSON 形式が正しい

## テストシナリオ 4: 翻訳キー欠落のフォールバック
**目的**: 欠落時に既定文言が表示されることを確認

**手順**:
1. ある言語ファイルのキーを一時的に削除。
2. 該当言語に切替し該当欄を確認。

**期待結果**:
- 既定文言が表示され、空欄やエラーがない

## テストシナリオ 5: グラフ文字
**目的**: 中文/日本語のグラフ文字が文字化けしないことを確認

**手順**:
1. 出勤統計ページを開く。
2. 期間を選択してグラフを生成。
3. 言語を切替してラベルを確認。

**期待結果**:
- グラフ文字が正常表示
