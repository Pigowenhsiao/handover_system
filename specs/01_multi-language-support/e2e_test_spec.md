# 多語言功能端到端測試規範

本系統為單機桌面應用，測試以手動流程為主。

## 測試場景 1: 語言選擇與切換
**目標**: 驗證語言可切換且即時更新

**步驟**:
1. 啟動應用程式，確認預設語言為日文。
2. 在登入畫面切換為中文。
3. 確認登入畫面文字即時更新。
4. 登入後切換為英文。
5. 確認主畫面各區文字即時更新。

**預期結果**:
- 語言切換成功且不崩潰
- 主要標題/按鈕/表頭顯示正確語言

## 測試場景 2: 切換時保留狀態
**目標**: 驗證語言切換不影響當前輸入

**步驟**:
1. 進入出勤頁面，輸入部分數值。
2. 切換語言。
3. 確認輸入值未被清空。

**預期結果**:
- 輸入狀態保持
- 文案更新

## 測試場景 3: 翻譯資源匯入/匯出
**目標**: 驗證管理員可匯入/匯出翻譯

**步驟**:
1. 以管理員身份進入「翻譯資源管理」。
2. 匯入一份 JSON 檔。
3. 匯出並檢查檔案內容。

**預期結果**:
- 匯入/匯出成功
- JSON 格式正確

## 測試場景 4: 缺少翻譯鍵回退
**目標**: 驗證缺鍵時使用預設文字

**步驟**:
1. 暫時移除某語言檔的單一鍵值。
2. 切換至該語言並觀察該欄位。

**預期結果**:
- 顯示預設文字，不出現空白或錯誤

## 測試場景 5: 圖表文字
**目標**: 驗證中文/日文圖表文字不亂碼

**步驟**:
1. 進入總結統計頁面。
2. 選擇日期區間並產生圖表。
3. 切換語言觀察圖表標籤。

**預期結果**:
- 圖表文字顯示正常
